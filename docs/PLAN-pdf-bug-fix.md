# PLAN: PDF Generation & Encryption Fix

This plan addresses the two critical bugs in the HRMS PDF generation system:
1. `blobStream is not defined` error.
2. Blank/corrupted PDF output when encryption is enabled.

## User Review Required

> [!IMPORTANT]
> I propose migrating the encryption logic from **PDFKit Native** to **pdf-lib**. PDFKit's native browser-side encryption is notoriously unstable due to crypto polyfill requirements. `pdf-lib` is the industry standard for reliable browser-side PDF encryption.
